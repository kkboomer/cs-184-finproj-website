Rendering pipeline design
Baseline
Project starts from HW3 path tracing (pathtracer, BSDFs, etc.).

Spectral representation
You introduce spectrum helpers and move thin-film and BSDF shading toward ColorVector instead of Vector3D so wavelength-dependent effects (thin film) integrate consistently across BSDFs.

Thin-film BSDF
You add ThinFilmBSDF with Airy interference, wired to ColorVector and to whatever Collada/XML supplies for film parameters (later thinFilmBsdf logic evolves in bsdf.cpp).

Path throughput
The tracer is refactored so ColorVector travels through bounces and RGB is produced only at raytrace_pixel. That keeps the interior of the path spectral until display conversion.

Sampling / GI
sample_f (BSDF sampling) is revised alongside path tracing tweaks. Global illumination for environment lighting is refactored so the path tracer uses the sampled environment direction directly instead of an older indirection pattern—cleaner coupling between env sampling and continuation rays.

Color science at the lights
ColorVector gains const usage and wavelength-related parameters tuned for lights. Later changes add CIE-oriented handling, XYZ, spectrum initialization for lights, and illumination paths adjusted so emitted light matches your spectral pipeline.

Lights and integration
A major refactor trims pathtracer.cpp, introduces light_info.h, and threads a uniform notion of light data through ambient / area / directional / point / spot so the tracer receives consistent spectral/light inputs without duplicated glue code.

Ongoing BSDF/spectral polish
Later edits continue to adjust bsdf.cpp and color_vector.cpp (thin-film interaction and color conversions), consistent with tightening spectral correctness rather than structural rewrites.

DAE files and Collada/scene design
Parser
The XML/Collada parser is updated so scenes can describe your thin-film BSDF and related parameters.

Scene progression
DAEs accumulate bubble-themed setups: single bubble in a room, multi-bubble/bunny variants, gemstone bubble (CB_gems_bubble), dragon-with-film (CBdragon_film), Lambertian spheres with film (CBspheres_lambertian_film), and iterative edits to n_bubble_ball / n_bubble_bunny.

Authoring intent in .dae
Commits explicitly tune n_bubble_bunny: diffuse color for consistency across materials and light color set to black where the authored lighting model should match reference renders (avoiding unintended illumination from that source).
